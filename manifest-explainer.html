TinyCloud Manifest Flow

Conversation Sync uses one manifest file to define the app namespace, default user capabilities, and the backend delegation template. The backend serves a runtime copy with its live DID inserted, and the frontend signs in from that manifest.

Source: examples/conversation-sync/manifest.json Runtime endpoint: GET /api/manifest Namespace: com.tinycloud.conversation-sync

Current Manifest

The file omits prefix and defaults. That is intentional: prefix defaults to id, and defaults defaults to true. 

{
  "id": "com.tinycloud.conversation-sync",
  "name": "Conversation Sync",
  "description": "Sync Fireflies and Google Meet transcripts into a TinyCloud space.",
  "permissions": [
    {
      "service": "tinycloud.hooks",
      "space": "default",
      "path": "sql/com.tinycloud.conversation-sync/conversations/conversation",
      "actions": ["subscribe"],
      "skipPrefix": true
    }
  ],
  "backend": {
    "name": "Conversation Sync Backend",
    "expiry": "7d",
    "permissions": [
      {
        "service": "tinycloud.kv",
        "space": "default",
        "path": "/",
        "actions": ["get", "put", "del", "list", "metadata"]
      },
      {
        "service": "tinycloud.sql",
        "space": "default",
        "path": "conversations",
        "actions": ["read", "write"]
      }
    ]
  }
}

Runtime Flow

The backend does not maintain a second permission list. It reads the manifest, injects its DID into the backend template, and returns a normal TinyCloud manifest with a real delegations[*] entry. 

sequenceDiagram
  participant File as manifest.json
  participant Backend
  participant Browser
  participant SDK as TinyCloud SDK
  participant Wallet

  Browser->>Backend: GET /api/manifest
  Backend->>File: read single manifest file
  Backend->>Backend: inject live backend DID
  Backend-->>Browser: runtime manifest with delegations[0]
  Browser->>SDK: createAndSignIn({ manifest })
  SDK->>Wallet: SIWE recap with defaults and backend delegation
  Wallet-->>SDK: signed session
  Browser->>SDK: resolveManifestDelegationPermissions(manifest, backendDid)
  Browser->>SDK: delegateTo(backendDid, resolved permissions)
  SDK-->>Browser: serialized UCAN, no extra prompt
  Browser->>Backend: POST /api/delegations
  Backend->>SDK: useDelegation(serialized UCAN)

Capability State

App-relative paths are resolved before signing. The backend then uses relative KV keys and the resolved SQL database name. 

stateDiagram-v2
  [*] --> ManifestTemplate
  ManifestTemplate --> RuntimeManifest: backend inserts DID
  RuntimeManifest --> SignedSession: wallet signs recap
  SignedSession --> DerivedDelegation: delegateTo resolved permissions
  DerivedDelegation --> BackendActive: backend useDelegation
  BackendActive --> UserKV: kv prefix com.tinycloud.conversation-sync
  BackendActive --> UserSQL: db com.tinycloud.conversation-sync/conversations
  BackendActive --> Expired: expiry reached
  Expired --> RuntimeManifest: sign in again

Resolved Paths

KV

Manifest backend permission path: "/" resolves to com.tinycloud.conversation-sync/. Backend code writes app-relative keys such as config/fireflies-key and transcript/{id}.

SQL

Manifest backend permission path: "conversations" resolves to com.tinycloud.conversation-sync/conversations. Backend SQL code calls access.sql.db(resolvedName).

Hooks

The hook grant authorizes the meta-resource hooks/sql/com.tinycloud.conversation-sync/conversations/conversation. It uses skipPrefix: true because hook scopes include the watched service and the already-resolved SQL table path.

Live Hooks

Live write-event hooks are disabled by default with VITE_ENABLE_TINYCLOUD_HOOKS=false. The hosted node currently advertises no hooks feature and returns 404 for POST /hooks/tickets, while a local TinyCloud node with hooks enabled exposes the endpoint. Turn the flag on only when the selected TinyCloud host supports hook tickets.

Manifest Options

Option Default Use it for
id Required App identity and default namespace prefix.
prefix id Omit for normal boilerplate isolation. Set "" only when paths should be global or already fully scoped. Set a custom string to migrate namespace.
defaults true true grants standard KV, SQL read/write, and capabilities read under the prefix. false disables automatic entries. "admin" adds SQL DDL and capability admin. "all" adds every supported service.
permissions Empty Add app runtime needs beyond defaults, such as tinycloud.hooks.
backend.permissions Required by this boilerplate App-relative permissions the backend needs. The backend turns these into a real manifest delegation by inserting its DID at runtime.
skipPrefix false Opt a permission out of prefixing for shared or meta resources. The hook grant uses this because its path already contains the resolved SQL path.
expiry 30d Sets the session default. A backend delegation can override it with backend.expiry.
includePublicSpace true Controls public-space companion capabilities when the SDK prepares the session.

Boilerplate Rule

Keep paths app-relative in the manifest. Let the SDK resolve the namespace. Backend code should derive full service paths from the manifest when the service does not apply a delegated prefix itself, such as named SQL databases and backend-owned KV bookkeeping.
