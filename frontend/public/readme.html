v0.4 - TinyCloud architecture

How Listen signs in, delegates, and syncs.

Listen is a transcript workspace where the browser owns TinyCloud sign-in, source setup, and sensitive secret writes, while the backend receives delegated capability to import, sync, normalize, and persist conversation data.

01 - roles

Each side gets a narrow job.

The user grants app capabilities during sign-in, then signs a portable delegation so the backend can operate without holding the wallet.

- browser

Wallet and Secrets owner

Loads the manifest, signs in with OpenKey, stores the Fireflies key, starts Google OAuth, uploads or pastes transcripts, and reads the encrypted secret vault item through the backend's decrypt grant.

- backend

Delegated sync worker

Verifies sessions, stores portable delegations by wallet address, activates delegated access, reads granted secrets, and runs provider sync and manual import routes.

- tinycloud

Data and capability boundary

Holds user app SQL/KV data and the Secrets vault. Webhook subscription, pending, and failed-event bookkeeping is stored through backend-owned KV.

This follows frontend/src/App.tsx, the local app manifest, cached /api/server-info, /api/workspace-state, direct TinyCloud reads, backend-owned Fireflies job state, Google Meet sync routes, and the manual transcript import endpoint.

03 - state

Workspace, source, and secret states.

The UI gates the inbox on source availability, existing conversations, delegation status, manual imports, and whether provider credentials can be used by the backend. Google Meet appears only when Google OAuth is configured.

Backend access

Policy source

/api/server-info returns the backend DID, expiry, and requested TinyCloud permissions. The frontend resolves them against the manifest before creating a portable delegation.

Validation

POST /api/delegations deserializes the delegation, checks it covers the current backend policy hash, activates it, stores it by wallet address, and caches delegated access.

Expiration

GET /api/workspace-state returns delegation status, backend secret readability, Google Meet connection state, and whether existing conversations are available. Expired or stale-policy delegations are removed before backend-only workflows continue.

04 - Fireflies secret

The backend gets a grant, not the raw write path.

The browser writes the secret directly to TinyCloud, then shares it with the backend DID. The backend only probes for access and never accepts raw Fireflies key writes.

Secret access states

No key

The app asks the user to paste a Fireflies API key. Only the frontend calls tcw.secrets.put("FIREFLIES_API_KEY", value).

Key saved

The key exists in TinyCloud Secrets, but the backend may still be unable to read it. The app shows "Finish Fireflies access" when backend delegation is missing.

Shared

The browser delegates per-secret KV read plus tinycloud.encryption/decrypt on the user's default network. After that, delegated backend routes can call access.secrets.get().

Sync ready

Fireflies sync can list transcripts, dedupe by source_id, write conversation rows to SQL, and write transcript blobs to KV.

05 - spaces, KV, SQL

Where data lands in TinyCloud.

The manifest app id is xyz.tinycloud.listen. Defaults create the app space. The browser uses the local app manifest for sign-in, then composes reachable delegatees from /api/server-info. The backend policy is limited to backend-specific KV/SQL/secret-read permissions plus the separate default-network decrypt grant. Backend-owned operational state, including webhook queues and Fireflies job progress, lives under the backend identity's xyz.tinycloud.listen/ KV namespace. That composition requires an SDK release that already supports tinycloud.encryption manifest entries. The backend SQL helper opens xyz.tinycloud.listen/conversations rather than the SDK default database.

tinycloud.sql

xyz.tinycloud.listen/conversations contains the conversation and participant tables. Current server storage aliases databases by final path segment, so paths ending in /conversations must not be treated as isolated stores. Manual imports write conversation.source = "manual".

tinycloud.kv

User KV stores transcript/{id} blobs for Fireflies, Google Meet, and manual imports, plus config/google-tokens. Backend-owned KV stores webhook config, pending queues, and failed webhook events.

secrets

FIREFLIES_API_KEY is written by the browser and read by the backend only after the per-secret KV read and default-network decrypt grants exist.

tinycloud.hooks

When enabled, the app subscribes to conversation row writes so live inbox updates can follow TinyCloud SQL changes.
